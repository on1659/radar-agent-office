LAYER 02
가드레일 레이어
명시적 지시 없이 Claude가 행동하지 못하도록 차단
훅 기반 
# 1단계: check-intent.sh (UserPromptSubmit)
# 사용자 메시지에 명시적 행동 지시가 있는지 탐지

if echo "$MESSAGE" | grep -qE \
  '해줘|실행해|고쳐줘|만들어|켜줘|꺼줘|수정해|추가해|삭제해'; then
  echo "ACTION_AUTHORIZED=true" >> "$CLAUDE_ENV_FILE"
else
  echo "ACTION_AUTHORIZED=false" >> "$CLAUDE_ENV_FILE"
fi

# 2단계: guard-tools.sh (PreToolUse)
# 허가 없으면 모든 도구 실행 차단

if [ "${ACTION_AUTHORIZED}" = "false" ]; then
  echo '{"decision":"deny","reason":"명시적 지시 없음"}'
fi
✅ 통과 — ACTION 지시
"CLAUDE.md 수정해줘"
"환경변수 테이블 추가해줘"
"빌드 실행해줘"
🚫 차단 — 질문/대화
"이게 맞아?"
"어떻게 생각해?"
"개선점이 있을까?"
⚠️ 현재 제한: 한국어 키워드만 감지. "fix", "update", "create" 같은 영어 지시는 ACTION_AUTHORIZED=false로 처리됩니다.

# 저장 위치
~/.claude/projects/d--Work-vibe-radar-agent-office/memory/
├── MEMORY.md        # 인덱스 (항상 컨텍스트에 로드)
├── user_*.md        # 사용자 역할·선호·지식
├── feedback_*.md    # 교정된 행동 패턴
├── project_*.md     # 프로젝트 컨텍스트·결정
└── reference_*.md   # 외부 시스템 참조
타입저장 기준활용
user 역할, 선호, 기술 스택 파악 시 설명 방식, 코드 스타일 조절
feedback 교정("이렇게 하지 마") 또는 확인("이게 맞아") 시 같은 실수 반복 방지
project 결정 사항, 배경, 마감 등 비코드 정보 제안 시 프로젝트 맥락 반영
reference 외부 시스템 위치 파악 시 정보 검색 위치 즉시 파악

컨텍스트 자동 조립

🔄
PDCA 단계 확인
bkit이 현재 DO 단계임을 인식 → 설계 문서 기반으로 구현 범위 판단
🔒
PreToolUse 훅 검사
도구 실행 직전 guard-tools.sh 재확인 → ACTION_AUTHORIZED=true 확인 후 실행 허가

    }],
    "PreToolUse": [{               // 모든 도구 실행 직전
      "matcher": ".*",               // 전체 도구 대상

      "command": "bash .claude/hooks/guard-tools.sh"

  }
}
